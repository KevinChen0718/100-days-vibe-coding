# 夢遊先生 · Mr. Sleepwalker

> 100 天 Vibe Coding 馬拉松 · **Day 1**
> 向 1999 年 Sarbakan 經典 Flash 解謎遊戲《Good Night Mr. Snoozleberg》（中文俗稱「夢遊先生」）致敬的重新詮釋作品。

夜深了，夢遊先生閉著眼睛從床上爬起，雙手前伸、慢吞吞又搖搖晃晃地一直往前走。
你**不能直接控制他**——只能在他走過去之前，**移動場景裡的物件**鋪出一條安全的路：
別讓他摔太重而驚醒、別讓他踩到尖刺或掉進水裡，把他平安送到出口。

配上即時生成的慵懶夜曲爵士，整體就是一片睡意朦朧的夜。

![遊戲畫面](assets/screenshot.png)

## 立即遊玩

直接用瀏覽器打開 `index.html` 就能玩，不需要任何安裝或伺服器。

```bash
open index.html        # macOS
```

## 玩法

- **放他走 ▶**：先生開始夢遊（也可以先佈置好物件再放他走）。
- **拖曳物件**：滑鼠或手指拖動場景中的道具。走路途中也能即時移動，靠的是「時機」。
- **重來**：把先生放回床上、重新佈置。
- **提示**：顯示物件該擺放位置的虛影。
- 鍵盤：`空白鍵` 放他走 / 下一關，`R` 重來，`H` 提示。

### 道具

| 道具 | 作用 |
|:--|:--|
| 木板 | 補地板缺口、搭橋 |
| 箱子 | 墊在落差中間當踏腳石；夠高時當牆讓他轉身 |
| 彈簧床 | 當跳板，把他往前上方彈飛，越過尖刺坑 |
| 床墊（雨棚）| 軟著陸，接住任何高度的墜落不受傷 |

另外場景裡會有**傳送門**（會發光的窗／下水道口）：走進去，會從連結的另一扇門冒出來——可以鑽下樓、繞過高牆、上到另一層，做出立體的進出路線。

### 失敗

摔落超過安全高度（會驚醒）、踩到尖刺、掉進水裡。失敗就重新佈置再試一次——本作刻意加了「每關獨立、隨時重來」，避開原作「一失誤整章重來」的勸退設計。

## 關卡

四個章節、共 **12 關**，難度循序漸進：前段一關引進一個新機關，後段開始組合、加入傳送門做立體路線：

1. 半夜・起床（鋪木板過洞）
2. 走下床台（墊箱子化解落差）
3. 轉個彎（用箱子當牆讓他轉身）
4. 彈跳板（彈簧床飛越尖刺）
5. 跳下雨棚（床墊軟著陸）
6. 穿牆而過（傳送門教學）
7. 煙囪滑梯（傳送下樓＋跨尖刺）
8. 一波三折（兩塊木板＋一個箱子）
9. 繞道而行（先轉身、搭橋、再鑽下水道）
10. 雙重門（傳送上樓＋上下兩道缺口）
11. 跳過水溝（彈簧＋木板）
12. 回到床上（終章：木板＋床墊＋木板＋傳送門，最長的一段）

> 原作有 8 集約 40 個場景，但精確的關卡設計只有部分被攻略記錄保存下來。
> 本作**忠實復刻核心玩法機制**，關卡則是依照原作第一章「城市屋頂」主題重新設計的致敬版本，並非逐關照搬。

## 技術

- 純前端、零依賴：HTML5 Canvas + 原生 JavaScript（無框架、無建置步驟、無外部圖檔，全部用 Canvas 向量繪製）。
- 物理引擎與繪圖、UI 分離：`js/engine.js` 是不碰 DOM 的純運算，因此可在 Node 直接跑自動化測試。

### 檔案結構

```
day-001-sleepwalker/
├── index.html
├── css/style.css
├── js/
│   ├── engine.js    # 物理引擎（純運算，Node 也能跑）
│   ├── levels.js    # 關卡資料（含每關解法座標）
│   ├── sprites.js   # Canvas 繪圖
│   ├── music.js     # WebAudio 即時生成的夜曲爵士背景樂
│   └── main.js      # 遊戲迴圈、輸入、UI、存檔、音效
└── test/
    ├── solve.js     # 自動驗證每關都解得開
    └── robust.js    # 量測每關擺放容差（玩家容錯空間）
```

### 自動化測試

物理引擎是純運算，所以可在 Node 直接驗證——不用真的手動玩每一關：

```bash
node test/solve.js     # 套用每關解法座標，模擬到通關 → 12/12 PASS
node test/robust.js    # 物件偏移多少還能過 → 每關最小容差 ≥ 12px
```
